# 测试与验收文档

## 1. 目标

本说明书定义本地 RAG 项目的测试范围、验收标准和执行规则。

独立测试执行 SOP 见：

- [sops/01-test-execution-sop.md](sops/01-test-execution-sop.md)

## 2. 测试范围

必须覆盖的测试范围包括：

- 后端单元测试
- 后端集成测试
- 前端单元测试
- 端到端测试
- 手工功能验证
- 可解释性验证
- 本地部署验证

## 3. 后端测试覆盖

后端测试必须覆盖：

- 认证成功与失败
- 文档上传校验
- 文档元数据持久化
- 索引任务生命周期状态变化
- chunk 生成规则
- 检索结果排序
- trace 持久化
- citation 持久化
- 设置读取与更新

## 4. 前端测试覆盖

前端测试必须覆盖：

- 登录表单行为
- 文档上传页行为
- 索引状态刷新行为
- 聊天会话创建
- 流式答案渲染
- citation 点击行为
- 证据面板渲染
- 设置页保存行为

## 5. 端到端覆盖

端到端测试必须覆盖：

1. 用户登录
2. 文档上传
3. 文档索引
4. 提交聊天 query
5. 流式答案完成
6. citation 查看
7. trace 详情查看

## 6. 可解释性验收

只有每个已完成答案都具备可解释性输出时，系统才可验收。

必须满足的可解释性验收项：

- 答案中包含 citation 标记
- citation 标记能映射到持久化证据项
- 证据项展示 document name
- 证据项展示 chunk index
- 证据项展示 retrieval score
- 证据项展示 rerank score
- 证据项展示 evidence text

## 7. 功能验收标准

只有以下条件全部满足时，交付才可验收：

- 本地环境可以成功启动
- 用户可以成功登录
- 用户可以成功上传文档
- 用户可以成功启动索引
- 可以成功查询索引状态
- 用户可以成功创建聊天会话
- 用户可以成功提交 query
- 答案可以成功流式输出
- 最终答案可以成功持久化
- trace 详情可以成功持久化
- 可以成功查看证据项
- 用户设置可以成功更新

## 8. 失败场景验收标准

只有系统正确处理以下失败场景时，交付才可验收：

- 非法登录凭证
- 非法文件上传请求
- 不支持的文档输入
- 索引失败
- 检索失败
- 答案生成失败
- 数据库连接失败
- Redis 连接失败

## 9. 数据验收标准

只有以下数据都被正确持久化时，交付才可验收：

- 上传文档
- chunks
- vectors
- 聊天会话
- 聊天消息
- trace 记录
- citation 映射

## 10. 性能基线

本地验收基线要求：

- 后端健康检查接口正常响应
- 标准文本文件在本地稳定运行中可完成上传和索引
- 标准用户 query 在本地稳定运行中可返回完整答案
- 流式输出期间前端保持可响应

## 11. 测试交付物

测试交付必须包含：

- 自动化测试结果摘要
- 手工测试结果清单
- 失败测试列表
- 已修复问题列表
- 验收签字记录
