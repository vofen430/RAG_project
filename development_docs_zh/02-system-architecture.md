# 系统架构设计说明书

## 1. 概述

系统采用本地部署方式，由前后端应用、一个关系型数据库、一个缓存服务以及一个本地文档存储目录组成。

系统包括以下部分：

- 一个 Vue 3 前端，用于文档管理、聊天交互、trace 查看和设置管理
- 一个 Spring Boot 后端，用于认证、文档接入、索引、检索、答案生成和 trace 持久化
- PostgreSQL 与 pgvector，用于业务数据和向量检索
- Redis，用于缓存、会话相关短期数据和限流数据
- 一个本地文件存储目录，用于保存上传的源文件

## 2. 运行时组件

### 2.1 前端

前端负责：

- 用户登录与会话初始化
- 文档上传与状态展示
- 聊天会话创建与消息展示
- 流式答案渲染
- citation 跳转
- 证据面板渲染
- 检索 trace 展示
- 用户设置操作

### 2.2 后端

后端负责：

- 认证与鉴权
- 文件上传处理
- 文档元数据持久化
- chunk 生成
- embedding 生成编排
- 向量持久化
- 相似度检索
- reranking 编排
- prompt 组装
- 流式答案输出
- citation 生成
- query trace 持久化
- 任务状态管理

### 2.3 PostgreSQL

PostgreSQL 是系统的主记录存储，负责保存：

- 用户账户
- 文档元数据
- 文档 chunks
- 向量 embeddings
- 索引任务
- 聊天会话
- 聊天消息
- 查询 traces
- citation 记录
- 用户设置

### 2.4 Redis

Redis 用于：

- access token 与 refresh token 的辅助短期数据
- 登录相关短期会话数据
- 请求限流计数
- 重复读取场景的缓存
- 短期索引进度缓存

### 2.5 本地文件存储

本地文件存储用于：

- 上传的源文件
- 临时导入文件
- 可选导出文件

## 3. 功能流转

### 3.1 文档接入流程

文档接入流程如下：

1. 前端上传文件。
2. 后端校验请求并将源文件保存到本地磁盘。
3. 后端在 PostgreSQL 中创建文档记录。
4. 后端创建索引任务记录。
5. 后端启动索引流水线。
6. 后端将 chunk 记录和向量数据写入 PostgreSQL。
7. 后端持续更新索引任务状态直到完成。

### 3.2 查询流程

查询流程如下：

1. 前端在聊天会话中发送用户问题。
2. 后端创建 query trace 记录。
3. 必要时后端先进行 query rewrite。
4. 后端在 PostgreSQL 中基于 pgvector 检索候选 chunks。
5. 后端对候选 chunks 做 reranking。
6. 后端组装最终 prompt 输入。
7. 后端调用模型供应商集成层。
8. 后端将答案流式返回前端。
9. 后端保存最终答案、citations 和 retrieval trace。

### 3.3 可解释性流程

可解释性流程如下：

1. 被召回的 chunks 获得稳定的 citation 编号。
2. 每个 citation 都映射到文档、chunk 和位置元数据。
3. 最终答案中包含 citation 标记。
4. 前端为该答案加载证据列表和 trace 详情。
5. 用户可以查看证据文本和检索元数据。

## 5. 持久化边界

持久化边界如下：

- 源文件保存在本地磁盘
- 结构化应用数据保存在 PostgreSQL
- 短期缓存和计数保存在 Redis

## 6. 安全边界

安全边界如下：

- 除登录和健康检查外，所有应用接口都需要认证
- 文档数据按用户所有权隔离
- 聊天会话按用户所有权隔离
- trace 记录按用户所有权隔离
- 供应商密钥和敏感配置不存放在源码中

## 7. 本地部署边界

本地部署边界如下：

- 一台本地机器
- 一个后端应用进程
- 一个前端应用进程
- 一个 PostgreSQL 实例
- 一个 Redis 实例
- 一个本地存储路径

## 8. 日志边界

后端必须记录：

- 认证事件
- 文件上传事件
- 索引生命周期事件
- 查询生命周期事件
- 检索结果数量
- rerank 结果数量
- 错误事件

前端必须记录：

- 登录失败
- 上传失败
- 流式中断
- trace 加载失败
